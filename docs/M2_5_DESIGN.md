## 1. 扩大地图 + 相机跟随（土豆兄弟/吸血鬼幸存者式）

### 问题
当前房间 = 窗口大小（1920×1080），一个画面看完全局，没有"探索/移动"的感觉。

### 方案
- **地图扩大到 ~3000×2000**（约 1.5× 窗口），英雄不在画面正中时相机跟随。
- 英雄始终居中显示（相机跟随英雄），房间边界用墙壁/Line2D 标示。
- 敌人仍从房间边缘刷出追击英雄（逻辑不变，只是房间更大）。
- `game_manager` 里 `room_rect` 改为更大尺寸；`main.tscn` 的 `RoomBorder` 同步扩大。

### 改动文件
| 文件 | 改动 |
|------|------|
| `main.tscn` | 添加 `Camera2D` 节点（follow hero）；RoomBorder 扩大到 3000×2000；Background 扩大 |
| `game_manager.gd` | `room_rect` 改为 `Rect2(80, 80, 2840, 1840)`；每帧设置相机位置 = 英雄位置 |
| `hero.gd` | 无改动（移动逻辑不变，房间更大而已） |
| `enemy.gd` | 无改动（追击逻辑不变） |

### 相机细节
- 用 Godot `Camera2D` 节点，`position_smoothing_enabled = true`（平滑跟随）。
- 相机挂在 GameManager 下，每帧 `camera.global_position = hero.global_position`。
- 或者更简单：Camera2D 直接作为 Hero 的子节点（自动跟随）。**推荐这个**——零代码。

## 2. 波次倒计时

### 问题
当前波次开始后无限打，直到所有怪清完。没有时间压力，节奏偏被动。

### 方案
- 每波有 **倒计时**（来自 `WaveCurves.wave_duration(wave)`，已有数据：base 25 + per_wave 1 秒）。
- HUD 显示倒计时（`第 3 波 | 剩余 28s | 敌人 12`）。
- **倒计时归零 → 波次结束**（不管怪有没有清完），进入构筑大厅。
  - 这与土豆兄弟一致：撑过时限 = 过关，不是必须杀完。
- 清完所有怪也可以提前结束波次（奖励剩余时间的金币，鼓励高效输出）。

### 改动文件
| 文件 | 改动 |
|------|------|
| `game_manager.gd` | `_process` 里 WAVE_IN_PROGRESS 状态减计时；归零 → `_on_wave_finished()`；清完怪也 → `_on_wave_finished()` |
| `hud.gd` | 新增 `update_timer(seconds: float)` 更新倒计时显示 |
| `main.tscn` | HUD 添加 TimerLabel（波次标签旁边） |

### 波次结束逻辑
```
WAVE_IN_PROGRESS 状态下每帧：
  _wave_timer -= delta
  if _wave_timer <= 0 或 spawner 全清：
    → WAVE_CLEARED（进构筑大厅）
    剩余时间奖励 = ceil(剩余秒数) × 2 金
```

## 3. 弹道/击杀特效（正反馈）

### 问题
当前弹道是纯黄色小圆，击杀无反馈。打击感弱。

### 方案（轻量，不引入美术资源）
- **弹道拖尾**：projectile `_draw` 加一个朝运动反方向的渐变拖尾（`draw_line` 从当前位置到上一帧位置，半透明）。
- **击杀爆裂**：敌人死亡时画一个快速放大的圆环（0.2 秒动画，半径从 0→40，透明度 1→0）。
  - 实现方式：enemy `died` 时不立即 `queue_free`，而是进入 "dying" 状态播 0.2s 爆裂动画再 free。
- **受击闪白**：敌人 `take_damage` 时闪一帧白色（`_flash_timer`，`_draw` 里根据 timer 混合白色）。

### 改动文件
| 文件 | 改动 |
|------|------|
| `projectile.gd` | 加拖尾（记录上一帧位置，draw 半透明线段） |
| `enemy.gd` | 加 `_flash_timer`（受击闪白 0.08s）；died 时播爆裂动画再 queue_free |

> 这个做轻量版即可，后续有美术资源再换。不做粒子系统（Godot GPUParticles2D），先用 `_draw` 程序化绘制，保持零依赖。

## 5. Bug 修复：羁绊按钮金币不刷新

### 问题
买了第一个羁绊（30 金）后，`bonds_drawn` 从 0→1，下次成本应该 40 金，但 HUD 按钮仍显示"羁绊 (30金)"。

### 根因
`update_bond_cost()` 只在 `_start_next_wave()`（波次开始）调用一次。买羁绊后 `bonds_drawn++` 但 HUD 不刷新。

### 修复
`game_manager._on_lobby_confirmed()`（lobby 关闭后调用）里加 `_hud.update_bond_cost(_build.bond_draw_cost())`。

### 改动文件
| 文件 | 改动 |
|------|------|
| `game_manager.gd` | `_on_lobby_confirmed` 加一行 `_hud.update_bond_cost(_build.bond_draw_cost())` |

## 6. Tab 显示当前数值 + 已有羁绊

### 问题
战斗中按 Tab 目前打开的是 lobby 的技能/羁绊切换。玩家无法在战斗中查看自己的 build 面板（当前属性、已有羁绊列表）。

### 方案

- 区分：如果 lobby visible → Tab 切页；否则 → 开/关角色面板。

### 改动文件
| 文件 | 改动 |

| `hud.gd` | 新增 `toggle_character_panel()`；角色面板渲染（属性 + 羁绊 + 技能列表） |
| `game_manager.gd` | `_unhandled_input` 处理 Tab 键：lobby 关着时 → 切角色面板 |

## 7. 渐进式刷怪（土豆兄弟式，波次越大刷越快）

### 问题
当前 `spawn_interval = 0.8s` 固定不变。前几波 10 只 × 0.8s = 8 秒就刷完，而波次倒计时 26 秒——怪在前 8 秒集中涌出，后 18 秒干等。视觉上像"一次性刷出来"，且没有随波次加压的节奏感。

### 方案
**刷怪间隔 = 波次倒计时 / 敌人数量**，让怪物均匀分布在整个波次时长内。波次越大（怪更多 + 时长更长），间隔越短（刷更快），自然形成压力递增。

```
spawn_interval = wave_duration(wave) / enemy_count(wave)
```

示例（用现有数据算）：

| 波次 | 敌人数 | 波次时长 | 间隔（改前固定 0.8s） | 间隔（改后） | 效果 |
|------|--------|---------|---------------------|-------------|------|
| 1 | 10 | 26s | 0.8s | **2.6s** | 前期宽松，慢慢来 |
| 5 | 16 | 30s | 0.8s | **1.9s** | 中期渐快 |
| 10 | 23 | 35s | 0.8s | **1.5s** | 压力上来 |
| 15 | 30 | 40s | 0.8s | **1.3s** | 密集刷怪 |
| 30 | 53 | 55s | 0.8s | **1.0s** | 高波持续高压 |

> 前 3 波间隔 > 2 秒，怪一只一只慢慢来，给玩家熟悉走位的时间。高波逐渐压到 1 秒一只，配合更大的怪量形成"撑住"的压力。这与土豆兄弟的刷怪节奏一致。

### 额外细节
- **首只怪延迟 1 秒**（不是 0.1s 立刻刷），给玩家"波次开始了"的缓冲。
- 间隔下限 0.3s（防超高波同时刷太多导致卡顿）。

### 改动文件
| 文件 | 改动 |
|------|------|
| `wave_spawner.gd` | `start_wave` 时算 `spawn_interval = maxf(0.3, WaveCurves.wave_duration(wave) / WaveCurves.enemy_count(wave))`；首只延迟改为 1.0s |

> 注：这个改动依赖 #2（波次倒计时）先完成——因为 `wave_duration` 的意义是"撑住这么久"，倒计时和刷怪节奏是配套的。
