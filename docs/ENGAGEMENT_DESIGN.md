# 《Project Roguelike-TD》心流与留存设计（v0.1）

> 目标：回答"怎么让玩家玩了上头"。这份文档和 GDD 平级——GDD 管"玩什么"，本文档管"为什么会停不下来"。
> 解构对象：Balatro（小丑牌）、金铲铲、KK 平台英雄防守图、Survivor.io。
> 所有机制都落到：**是什么 → 心理原理 → 怎么落地 → 验收指标 → 对 GDD 的影响**。

---

## 0. 一句话原则

> **爽感不来自难度，来自"我的组合解开了谜题"的智力高潮。**
> 我们要做的不是"把数值做难"，而是把 **build 的因果关系做透明**，并不断制造"再来一把就能成"的诱惑。

---

## 1. 理论框架："上头"的 6 个齿轮

任何让人停不下来的游戏，几乎都同时转着这 6 个齿轮。先做自检：我们的塔防现在有哪几个？缺哪几个？

| # | 齿轮 | 心理原理 | 一句话 | 当前状态（已据决策更新） |
|---|---|---|---|---|
| 1 | **短期目标** | 可达性 + 即时反馈 | 每一刻都知道"下一步要干嘛" | ✅ M2 已实现（清波 + 波次进度条 + 英雄血条） |
| 2 | **变率抽奖** | 变率奖励（variable reward） | "下一次刷新会不会出神牌" | ✅ M2 已实现（3 选 1 + 刷新递增 + 羁绊池多样性） |
| 3 | **乘法堆叠** | 数字膨胀爽感 | ×2 ×3 ×1.5 滚雪球 | ✅ M2 已实现（卡片显示加成数值 + final 乘区 + elemental） |
| 4 | **赌徒代价** | 近失 + 主动承担风险 | "我借这笔债重投，能翻盘" | 🟡 near-miss 数值已校准（B-3）；诅咒/代价系统待 M3 装备 |
| 5 | **超短循环** | 快速试错 | 一局 15–25min，立刻"再来" | ✅ 设计已定（分章节 10 波 + 续局，决策 7） |
| 6 | **可见 combo** | 因果可见 + 仪式感 | "我能看见我的组合在咬合" | 🟡 卡片加成已显示（M2）；伤害逐段弹出/结算诊断待 M4 |

**结论（M2 后更新）**：6 个齿轮中 1/2/3 已在客户端落地，4/6 部分实现，5 设计已定。下阶段 M3（装备/诅咒）补齐赌徒代价，M4（联动）补齐可见 combo 的伤害可视化。

## 2. 机制清单（可执行、可验收）

### M1. 波次级短期目标——把"撑住"变成"可见进度条"

**问题**：现在每波的目标是"清光所有怪"，但玩家中途无法衡量"我还差多远 / 我够不够强"。塔防常见病是"前 20 秒无聊，后面突然崩"。

**落地**：
- 每波顶部一条**敌人血池进度条**（已击杀血量 / 总血量）。让玩家像 Balatro 看盲注分数线一样，时刻知道进度。
- **Boss 波单独显示 Boss 血条** + 一条"我方核心血量条"，形成对峙。
- 在波次进行中**实时预测**："当前 DPS 预计清波时间 ≈ 18s / 总时长 30s ⚠️ 可能清不完"——这是引导玩家构筑的隐性提示（不是强提示）。

**验收**：玩家在任意时刻能在 1 秒内回答"我这波能不能过、还差多少"。B-2 模拟器要输出"波次剩余血量曲线"，验证不会出现"前期无压力后期突然崩"的断层。

**对 GDD 影响**：GDD §3.3 的曲线需要附一张"玩家 DPS vs 所需 DPS"对比图（B-3 产出）。新增 §战斗 HUD 设计：进度条 + 实时 DPS 读数。

---

### M2. 构筑大厅 = 商店 + 刷新（**最关键的补强**）

**问题**：Balatro 上头的核心动作是"刷新商店找关键牌"。我们的 3 选 1 是**固定 3 张**，缺少这个动作 → 没有"再刷一次"的多巴胺循环。

**落地**：波次间进入"构筑大厅"，三类操作都可**花金币刷新**：

| 操作 | 基础 | 刷新成本 | 说明 |
|---|---|---|---|
| 抽技能（3 选 1） | 每清波 1 次免费机会 | 重投 10 金（每次 +3） | 让"攒钱=更多重投=更易凑齐 combo" |
| 抽羁绊（3 选 1） | 30 金/次 | 重投 8 金 | 同上 |
| 装备升级 | 按等级付费 | — | 保底轨不刷新（确定性是它的价值） |

**关键设计**：
- **重投成本递增**（10→13→16…），防止无限刷新破坏随机性。
- **重投次数软上限**：每波大厅最多重投 N 次（如 5 次），强制玩家在"刷出更好的"和"留着钱抽羁绊"之间取舍——这就是 Balatro 利息系统的等价物。
- **"锁定"功能**（pin）：花少量金币锁定 1 张牌不随刷新消失，给"差一张就成型"的玩家希望。

**为什么这是留存核武器**：它把"经济系统（装备层）"和"构筑（技能/羁绊层）"咬合成闭环——**有钱 = 更多选择 = 更强 build**。钱不再只是数字，是"可能性"。

**验收**：B-2 模拟器要输出"玩家平均重投次数 vs 通关率"曲线。若玩家几乎从不刷新（说明成本太高）或无限刷新（说明太便宜），都要调。

**对 GDD 影响**：
- GDD §3.4 经济表新增"重投成本"行（技能重投 10 金递增 / 羁绊重投 8 金递增）。
- GDD §4.3 抽取规则补充"重投与锁定"。
- GDD §6.1 羁绊抽取补充"刷新"。

---

### M3. 诅咒词条重设计——从"纯亏"到"诱人的代价"

**问题**：现在装备诅咒词条（伤害 -10%、攻速 -10%）是纯负面，抽到就是被惩罚。Balatro 的精髓是**玩家主动选择承担风险换收益**，被动惩罚不会上头，只会劝退。

**落地**：把诅咒改成"**代价换收益**"的双向词条。举几个具体设计：

| 词条 | 代价 | 收益 | 设计意图 |
|---|---|---|---|
| 血祭 | 受到伤害 +25% | 金币获取 +60% | 高风险高经济，滚雪球赌徒 |
| 焚天 | 每 5 秒掉 1% 核心血量 | 最终伤害 +20%（乘区） | 用生存换爆发，斩杀流核心 |
| 断舍离 | 羁绊池上限 -2 | 每个羁绊效果 +30% | 少而精的构筑，金铲铲式经营 |
| 厄运契约 | 抽取成本 +20% | 抽到传说概率 +50% | 赌狗专属 |
| 时之刃 | 攻速 -15% | 每次命中伤害 +40% | 慢速重击 build |

**关键**：每条诅咒都必须**同时有代价和收益**，让玩家在"要不要接这个 deal"上纠结。这才是赌博快感。

**验收**：B-3 蒙特卡洛要统计"携带诅咒的 build vs 不携带"的通关率——若诅咒 build 普遍更弱，说明收益不够诱人；若普遍更强，说明代价太低。

**对 GDD 影响**：GDD §5.2 装备词条池整张表重写为"代价-收益"结构。

### M4. 可见的 combo 链条——伤害数字逐段弹出

**问题**：现在伤害公式是黑盒，玩家只看到最终掉血。Balatro 上头的一大半来自计分时**数字一个个跳出来**的仪式感。

**落地**：每次命中，客户端把伤害公式分段动画弹出，对应 GDD §3.1：

```
230 (ATK×ratio)  →×1.60 (物伤)  →×1.25 (技能倍率)  →×1.15 (最终)  →×1.33 (暴击)  = 704
```

- 数字**逐段跳出**，每段 80–120ms，最后总和放大爆裂。
- **乘区越大，该段数字越突出**（暴击/最终伤害用金色特大字）。
- 多重射时每根弹**分别**弹出，强化"我的 build 在疯狂输出"的视觉。

**为什么有效**：玩家能**看见因果**——"哦原来这条 +15% 最终伤害让数字从 600 跳到 700"。这会驱动他们去**主动追求特定乘区**，而不是无脑堆数值。

**验收**：纯表现层，不需数值验证。但要在 playtest 时观察：玩家是否会盯着伤害数字看（说明它有效）。

**对 GDD 影响**：新增 §表现：伤害结算动画规范。这是客户端 spec，不是数值。

### M5. Near-miss（近失）——濒死体验 + "差一点就成"

**问题**：最容易让人点"再来一局"的不是胜利，是"差一点就赢"。要**故意制造**这种体验，但要保证它**公平**（玩家能复盘出"差在哪"）。

**落地（两条）**：

**(a) 濒死体验**：
- 核心血量在最后几波**反复濒死**（10%–25% 血量震荡），配红血闪烁 + 震屏 + 心跳音效。
- **不要**设计成"前期满血后期突然暴毙"（不公平），而是**全程贴着死亡线**——玩家始终觉得"我可能过不了"，但又一波波惊险过关。这是塔防版"心跳流"。
- 实现：让所需 DPS 和玩家 DPS 的曲线**长期接近但不交叉**，只在玩家 build 成型后才拉开。

**(b) 失败结算页的"差一点"**：
- 显示**具体差距**："还差 2,300 总伤害（≈ 1 个传说词条 / 或 1 个联动未触发）"。
- 显示**关键节点**："第 22 波时你的 DPS 为 1,200，所需 1,500——差 25%。建议优先堆 final 乘区或凑齐 XX 联动"。
- 这把失败从"运气差"变成"明确改进方向"，驱动再开一局。

**验收**：
- B-3 模拟器统计"玩家死亡血量分布"——理想是双峰：要么满血碾压（build 成型），要么 5%–20% 血量惜败（near-miss）。若大量玩家 50% 血量失败，说明难度断层。
- B-3 输出"失败时的差距分析"模板。

**对 GDD 影响**：GDD §9 节奏目标表新增"血量曲线设计"约束（贴线不死、成型后碾压）。新增 §结算页：差距诊断设计。

---

### M6. 可见的终极 combo——长线钩子

**问题**：Balatro 玩家会为了"凑出无限分 build"反复玩几百小时。你要在数据里**埋好几个传说级 combo**，并且**让玩家早期就知道它们存在**（种草）。

**落地**：
- 在羁绊/技能图鉴里，传说 combo 显示为**模糊剪影 + 一句诱人描述**（"凑齐遮天三件 + 天帝拳，触发未知的力量"），不剧透具体数值，激发追求欲。
- 玩家**首次触发**任意联动时，全屏特效 + 解锁图鉴——这是"啊哈时刻"的仪式化。
- 设计 **3–5 个 Tier-S 终极 combo**（如天帝之拳、大圣闹天），它们的存在本身就是长线追求。

**验收**：数据层确保每个传说 combo 都**可达成**（B-3 蒙特卡洛要验证触发率 >0，且不是必出）。

**对 GDD 影响**：GDD §7 联动清单要明确标注每个的 Tier（S/A/B），并新增 §图鉴/成就系统设计。

## 3. 节奏与时长（M5 的具体化）

Balatro 一局 15–40min。我们当前 30 波，按 GDD 算约 20–30min，**偏长**，尤其手机端。

**建议：分段式单局**：
- **一个"章节" = 10 波**（小 Boss 收尾），约 8–10min。一章 = 一段完整心流。
- **三个章节连续 = 一个完整 run**（30 波），但每章结束可**中途退出并保留进度**（下次继续），适配手机碎片时间。
- 长局 meta 奖励（解锁新羁绊池/新英雄）只在**完成完整 run**时给，激励一次性打完。

这样兼顾"碎片化可玩"和"深度构筑的完整体验"。验收指标：单局完整 run ≤ 25min。

**对 GDD 影响**：GDD §2 核心循环加入"章节分段点"；§3.3 曲线表标注每 10 波为分段节点。

## 4. 反模式清单（明确"不要做什么"）

这些常见做法会**杀死上头感**，必须规避：

| 反模式 | 为什么要避免 | 我们的对策 |
|---|---|---|
| **纯随机无策略** | 抽到啥用啥，玩家没有 agency → 无聊 | 所有抽选都允许重投/锁定（M2），把随机变决策 |
| **build 不透明** | 玩家不知道为什么强/弱 → 无改进动力 | 可见 combo 动画（M4）+ 结算差距诊断（M5） |
| **单局过长** | 失败成本高，不敢再开 → 留存崩 | 分章节（§3），25min 上限 |
| **惩罚 > 诱惑** | 抽到烂牌纯亏 → 劝退 | 诅咒改"代价换收益"（M3） |
| **难度断层** | 前期无聊后期暴毙 → 心流断裂 | DPS 曲线贴线设计（M5a） |
| **终极目标不可见** | 不知道追求什么 → 无长线动力 | 图鉴种草 + 传说 combo（M6） |
| **构筑与战斗脱节** | 选完就挂机看戏 → 参与感低 | 波次中可微操（手动放技能/走位），见下 |

## 5. 补一个易被忽视的齿轮：战斗中的微操参与

**问题**：英雄自动战斗是好的（移动端友好），但完全挂机会削弱参与感，玩家变成"看戏"。

**落地（轻量微操，可选其一）**：
- **手动大招**：每 X 秒蓄力一次的主动技能，玩家点击释放（清屏/爆发），关键时刻有掌控感。
- **走位**：玩家可拖动英雄走位，影响"吸引仇恨/集火方向"（不是操作负担，而是策略选择）。
- **目标优先级**：玩家可一键切换英雄攻击目标策略（最近/最脆/Boss）。

**优先级**：手动大招最容易做、收益最高。建议 M2 阶段就加入。

**对 GDD 影响**：新增 §战斗操作设计：手动大招（蓄力 → 释放）。

## 6. 心流自检清单（持续 review 用）

每次加新内容/改数值，过一遍这 8 条：

- [ ] 玩家任意时刻都知道"下一步目标是什么"吗？（M1）
- [ ] 玩家能通过花钱改变抽取结果吗？（M2 刷新）
- [ ] 每个"诅咒/代价"是否同时给了诱人收益？（M3）
- [ ] 玩家能否看见自己 build 的乘区如何咬合？（M4）
- [ ] 失败时玩家能否明确复盘"差在哪"？（M5）
- [ ] 是否存在 3+ 个 Tier-S 终极 combo 作为长线追求？（M6）
- [ ] 单局是否 ≤25min 且能碎片化分段？（节奏）
- [ ] 战斗中是否有至少一个玩家的主动操作点？（微操）

## 7. 优先级与路线图

按"对上头感的性价比"排序（性价比 = 爽感提升 / 开发成本）：

| 优先级 | 机制 | 开发成本 | 为什么这个排序 |
|---|---|---|---|
| **P0** | M4 可见 combo 动画 | 低（纯表现） | 性价比最高，几乎白捡 |
| **P0** | M2 商店刷新 | 低（经济系统已有，加刷新动作） | 补上最缺的齿轮，咬合闭环 |
| **P0** | M5b 结算差距诊断 | 低（UI） | 直接驱动"再来一局" |
| **P1** | M1 进度条 + DPS 读数 | 低 | 改善全程心流 |
| **P1** | M3 诅咒重设计 | 中（数据 + 平衡） | 改变构筑策略深度 |
| **P1** | 手动大招（§5） | 中 | 提升战斗参与感 |
| **P2** | M5a 濒死曲线调参 | 高（需大量模拟） | 依赖 B-3 模拟器 |
| **P2** | M6 图鉴 + 种草 | 中 | 长线留存 |
| **P2** | 分章节（§3） | 中（改循环结构） | 中后期再做 |

## 8. 对后续数值验证（B-2/B-3）的影响

这份心流设计**直接定义了 B-2/B-3 要验证什么**：

**B-2（经济 + 抽取池）必须新增**：
- 重投成本曲线 vs 玩家平均重投次数（M2 验收）。
- 重投对 build 成型速度的影响（攒钱刷 vs 不刷）。
- "每波 1.5–2 次操作"目标要包含重投动作。

**B-3（完整 Run 模拟 + 蒙特卡洛）必须新增**：
- 玩家死亡血量分布（M5 验收：理想双峰，5%–20% 惜败为主）。
- "差一点"差距分析输出（M5b 模板）。
- 传说 combo 触发率分布（M6 验收：可达成但不泛滥）。
- 携带诅咒 build vs 纯净 build 的通关率对比（M3 验收）。
- 手动大招对清波效率的贡献（§5 验收）。

**结论**：心流设计不是"做完游戏再调"，而是**从一开始就约束数值模型**。B-2/B-3 的模拟器要把这些指标当一等公民。

## 9. 决策记录（2026-07-03）

| # | 决策 | 选项 | 说明 |
|---|---|---|---|
| 1 | **商店刷新** | ✅ 接受 | 重投递增成本 + 每波软上限 + 锁定功能（见 M2） |
| 2 | **诅咒方向** | ✅ 代价换收益 | **核心要求：平衡到"难以取舍"**——代价与收益必须势均力敌，让玩家纠结（见 §10） |
| 3 | **微操** | ✅ 手动大招 | 蓄力 → 释放的主动爆发技能，关键时刻掌控感（见 §5） |
| 4 | **分章节** | ✅ 可中断保留 | 10 波一章节，可中途退出并保留进度，下次继续（见 §3） |
| 5 | **终极 combo 数量** | ✅ 首版 3 个 Tier-S | 作为长线追求钩子（见 M6 / GDD §7） |
| 6 | **碎片化** | ✅ 支持续局 | 一局可暂停、退出后回来继续，适配移动端碎片时间（见 §11 存档） |

> 决策 2（诅咒平衡）是重点设计目标：**不是设计出一个"最优诅咒"，而是让每条诅咒在特定 build 下都成立**——血祭适合滚雪球流、焚天适合斩杀流、断舍离适合精铺流。平衡标准见 §10。

## 10. 诅咒平衡的验收标准（决策 2 的展开）

"难以取舍"如何量化？给出可验证的标准：

| 指标 | 目标 | 说明 |
|---|---|---|
| **携带率分布** | 每条诅咒在一局中被携带的频率 15%–35% | 若某条 <10% 说明太亏（被嫌弃），>50% 说明太赚（必拿） |
| **build 相关性** | 每条诅咒至少有 1 种 build 让它成为"最优解" | 不能有任何诅咒在所有情况下都更差 |
| **通关率差异** | 携带诅咒的 build 通关率 ≈ 不携带（±10%） | 整体平衡，不偏向赌或保守 |
| **场景区分度** | 同一诅咒在不同 build 里效果差异大 | 断舍离在"羁绊少但强"build 是增益，在"堆数量"build 是负收益 |

这 4 条直接成为 B-3 蒙特卡洛模拟器的验收指标。平衡做不到时，调的是**收益数值**（而非取消代价），保持"赌博"张力。

## 11. 续局存档的技术影响（决策 4、6）

"可中断保留"对架构有实际影响，提前明确：

- **存档粒度**：RunState 序列化为 JSON，**在每个波次结束（进入构筑大厅时）写一次存档**——不是逐帧。
- **存档内容**：当前波次、英雄 build（技能+词条+等级）、羁绊池、已吞噬池、装备（等级+词条）、金币、核心血量、**RNG 种子 + 已消耗的随机序列**。
- **为什么存种子**：续局后所有随机必须与"如果一直玩下来"一致——这是种子化 RNG 的红利，也是**服务端校验续局合法性**的基础（防作弊改存档）。
- **跨设备**：存档走服务端同步（技术文档 §6 已决策需要服务端）。续局可在另一台设备继续。

**对架构的影响**：技术文档 §6 已覆盖（RunState 客户端跑 + 服务端存检查点 + 种子可复现）。无需新增服务，但存档 schema 要在 B 阶段定清楚。
