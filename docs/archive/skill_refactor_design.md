# 玩法重构：技能 = 体系入口（不再是抽取消费）

> ✅ **状态：已实施（2026-07-12）。** 数据 + 引擎重构完成（skills.yaml 精简为每体系 1 个起点技能；synergies.yaml 触发改 path_realm/bond_devoured_set；九秘转羁绊；draw_skill_offers 等已删除）。客户端 UI（体系选择面板、试炼关卡）为后续里程碑。
> 动机：现有技能系统与羁绊职责重叠（都往 CombatStats 加数值），体验鸡肋。
> 核心改动：技能从"每波抽取的消费"改造为"体系选择的入口"——选了技能才解锁对应羁绊池，技能随境界自动升级。
>
> **实施落地说明**（对照下方提案）：
> - skills.yaml：11 个技能 → 仅 emperor_fist（遮天起点技能）。✅
> - 九秘：`jt_*` 技能 → 遮天境界子羁绊 `zt_jm_*`（分散到 9 个境界，修满即集齐）。✅
> - synergies.yaml：触发 `skill_owned` → `bond_devoured_set` / `path_realm`。✅
> - draw_skill_offers / take_skill_offer / owned_skills / skill_level / skill_reroll：已删除。✅

## 0. 一句话定位

> **技能是体系的钥匙：选了遮天技能 → 解锁遮天羁绊池 → 修境界 → 技能自动变强 → 去试炼关卡挑战突破。**
> 体系数量不设上限，玩家自己决定选几个（但单修必败于试炼，多修互补才能过）。
> 首版做 3 个体系：遮天（物理）/ 星辰变（法术）/ 宠魅（召唤），其他后续补充。

## 1. 现状问题

| 问题 | 表现 |
|------|------|
| **技能与羁绊重叠** | 技能给 `physical_dmg`/`attack_speed`，羁绊也给同字段——玩家分不清区别 |
| **九秘尤其鸡肋** | `jt_xing`(攻速) 和 `zt_siji_left_leg`(攻速) 完全重复 |
| **技能缺独占价值** | 只剩 `atk_ratio`(倍率) + 联动触发器(skill_owned)，不够撑起一个系统 |
| **抽取决策单调** | 技能 3 选 1 变成"再抽一层数值"，没有玩法风格选择 |

## 2. 新设计：技能 = 体系入口

### 2.1 体系选择（开局 / 大厅）

玩家在游戏开始（或第一次进大厅）时，**选择 1-4 个体系的起点技能**：

| 体系 | 起点技能 | 选择后解锁 |
|------|----------|-----------|
| 遮天 | 天帝拳 | 遮天羁绊池（轮海→天帝）+ 九秘解锁机制 |
| 星辰变 | 星辰变功法 | 星辰变羁绊池（星云→宇宙）+ 星辰法术 |
| 宠魅 | 召唤·莫邪 | 宠魅羁绊池（魂徒→万年）+ 召唤机制 |
| （第4体系待定） | ... | ... |

**规则**：
- 每个体系 **1 个起点技能**（不是多个）
- 选了技能 → 该体系羁绊进卡池；**没选 = 羁绊永远不出现**
- 技能槽 = 体系数（选 4 个体系 = 4 个技能槽全满）
- 建议同时选 3-4 个（互补：物理+法术+召唤+生存）

### 2.2 技能随境界自动升级

技能不再是独立抽取/升级的对象，而是**境界的附属 reward**。每次突破境界，起点技能自动获得强化：

```
遮天·天帝拳 升级路线（随境界自动）：
  选了天帝拳（开局）     → 基础：物理伤害 ×2.0
  修到化龙（境3）        → +附带 10% 真伤
  修到大帝（境6）        → +附带 30% 真伤（圣体破诅咒）
  修到天帝（境8）        → 肉度 +100%（天帝之躯）
```

```
星辰变·星辰变功法 升级路线：
  选了（开局）           → 基础：法术伤害 ×1.5
  修到恒星（境5）        → +AOE 范围
  修到黑洞（境7）        → +吞噬（击杀回血）
  修到宇宙（境8）        → +最终伤害 ×2.0
```

```
宠魅·召唤莫邪 升级路线：
  选了（开局）           → 基础：召唤月光狐
  修到魂师（境2）        → 莫邪进化为六尾（伤害+50%）
  修到魂宰（境5）        → 解锁白魇魔（第二召唤兽）
  修到魂朽（境6）        → 半魔化变身（周期性爆发）
  修到万年（境8）        → 莫邪=幽冥狐尊·紫帝（终极）
```

> **实现**：技能升级 = 境界 reward 里追加技能强化 effect。不需要单独的技能升级 UI/操作。`realm.reward` 里加 `skill_upgrade: {id, effect}` 字段。

### 2.3 试炼关卡（独立，波外）+ 爽点演出

**境界突破不再自动发生**——集齐当前境界的羁绊后，玩家要**去打一场独立试炼关卡**才能突破。

**试炼设计**：
- 试炼是**完全独立的关卡**（离开正常波次序列，有自己的场景/规则）
- 每个境界的试炼难度递增（轮海试炼简单，天帝试炼极难）
- **试炼难度按"单体系战力曲线"设计**——单修 1 个体系的战力**必然打不过中后期试炼**
- 多修互补（物理输出+法术 AOE+召唤扛伤+生存续航）才能过

**为什么单修必败**：
- 单修遮天：物理强但缺 AOE（被群怪淹没）/缺续航（没吸血）/缺控制
- 单修星辰变：法术 AOE 强但身板脆（被 Boss 秒）
- 单修宠魅：召唤多但主人弱（召唤被清完就输）
- **多修互补**才能覆盖所有短板

**失败后果**：试炼输了 = 突破失败，羁绊不消失，可以攒钱/升装备后再试（不卡死进度，但要付出时间成本）。

**突破爽点演出**（玩家打赢试炼后）：
- **AI 生成插画 + 文字旁白**：每个境界的突破配一张专属插画（用现有 CogView+GLM-Image 美术管线生成），配文字旁白营造爽点。
  - 例：遮天修到大帝 → 插画"叶凡圣体大成" + 旁白文字"**圣体大成了！**"
  - 例：星辰变修到恒星 → 插画"秦羽化身恒星" + 旁白"**恒星境成！法则初显！**"
  - 例：宠魅修到魂朽 → 插画"楚暮半魔化" + 旁白"**半魔觉醒！万兽臣服！**"
- **可跳过**：玩家可点击跳过演出，直接进入下一境界（不强制观看）。
- **素材来源**：复用 `docs/ART_PIPELINE.md` 的 CogView（草图）+ GLM-Image（精修）两步管线，按境界批量生成突破插画。先做遮天 9 张（轮海→天帝），后补星辰变/宠魅。
- **音频**：后期补充（TTS 合成突破台词，或音效），首版只做插画+文字。

### 2.4 联动机制调整

现有联动用 `skill_owned` 触发——新设计下起点技能天然满足这个条件（选了体系就有技能）。联动改为：

| 旧触发 | 新触发 | 说明 |
|--------|--------|------|
| `skill_owned: emperor_fist` | `path_realm: {zhutian: >=6}` | 修到大帝境触发（技能=体系入口，有技能=在修） |
| 九秘合一（9个 skill_owned） | 九秘从技能改成**遮天境界子羁绊**，集齐触发 | 九秘归入遮天 path 的子羁绊 ✅ 已实施（`zt_jm_*`） |
| `skill_tag: thunder` | `bond_set: xingchenbian` + realm | 按体系+境界触发 |

> 联动触发器从"拥有某技能"改成"某体系修到某境界"——更直观，且不需要额外追踪技能。
>
> **九秘安置**：从技能改成遮天的**境界子羁绊**。九秘分散到各境界（突破时随机解锁一个，沿用现有 `unlock_jt_secret` 机制），作为子羁绊参与"集齐突破"，不再是独立技能。
>
> **实施落地（2026-07-12）**：九秘已落地为遮天境界子羁绊 `zt_jm_jie / zt_jm_dou / ...`，分散到遮天 9 个境界。修满遮天 9 境界即自动集齐全部九秘羁绊，触发"九秘合一·神禁"联动（synergy: `zhutian_nine_secrets`，触发条件 `bond_devoured_set: zhutian`）。原 `unlock_jt_secret` 的"随机解锁"设想已改为"每境界各分配一个九秘子羁绊"，见 `balance/data/bonds.yaml`。

## 3. 玩法循环（重构后）

```
开局：选 3-4 个体系（拿 3-4 个起点技能）
  ↓
大厅：抽羁绊（只能抽已选体系的羁绊池）
  ↓ 集齐当前境界羁绊
试炼关卡：打突破战（独立关卡）
  ↓ 赢了 → 突破 → 技能自动升级 + 解锁下一境界
  ↓ 输了 → 攒钱/升装备，再来
重复 → 修满多个体系 → 终极联动
```

**对比旧循环**：
- 旧：每波抽技能 3 选 1 + 抽羁绊 3 选 1（两个抽取环节，技能那层鸡肋）
- 新：开局选体系（1 次决策）+ 每波抽羁绊（1 个抽取环节，更聚焦）+ 试炼（主动玩法）

## 4. 对现有系统的影响

### 4.1 删除/改造的部分

| 现有系统 | 处理 | 影响 |
|----------|------|------|
| `draw_skill_offers`（技能 3 选 1） | **删除** ✅ | lobby 的技能 tab、技能抽取 UI 全删 |
| `take_skill_offer` / `owned_skills` | **改造** ✅ | 改为 `chosen_paths`（已选体系列表），起点技能自动赋予 |
| 技能的 `base_affixes` | **删除** ✅ | 技能不再直接给 affixes，改由境界 reward 强化 |
| `skill_level`（技能升级机会） | **删除** ✅ | 技能随境界自动升级，不需要手动升级 |
| 九秘（`jt_*` 技能） | **改造** ✅ | 从技能改成羁绊（归入遮天境界子羁绊 `zt_jm_*`） |
| `skill_owned` 联动触发 | **改造** ✅ | 改成 `path_realm` / `bond_devoured_set` 触发 |

### 4.2 新增的部分

| 新系统 | 说明 |
|--------|------|
| **体系选择 UI** | 开局/大厅选 1-4 个体系（卡片展示各体系定位） |
| **起点技能自动赋予** | 选了体系 → 自动获得起点技能（进 owned_skills） |
| **境界 reward 里的 skill_upgrade** | 突破时自动强化对应起点技能 |
| **试炼关卡** | 独立场景，集齐羁绊后可挑战，赢了突破 |
| **羁绊池按体系过滤** | draw_bond_offers 只给已选体系的羁绊 |

### 4.3 保留不变的部分

| 系统 | 说明 |
|------|------|
| 羁绊阶梯抽取（轮海→天帝） | 核心玩法不变 |
| 境界突破 reward | 不变（追加 skill_upgrade） |
| 装备系统（升级+词条） | 不变 |
| 联动引擎 | 框架不变，触发条件改 |
| CombatStats / EffectResolver | 不变 |
